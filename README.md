# 自然语言处理基础实践项目 (FudanNLP)

本项目系统记录了自然语言处理基础任务的实验过程与核心结论。内容涵盖现代大模型推理框架对比、基础机器学习算法、深度学习分类模型以及 Transformer 架构的完整实现。

## 项目结构总览

NLP-Tasks 根目录按照任务阶段划分为四个核心模块，用户可通过以下链接快速跳转至对应实验的详细报告：

1. [任务零：熟悉 WSL 环境与 vLLM 高效推理](Task0/README.md)
   包含 vLLM 与原生 Transformers 框架的吞吐量对比实验及性能优化原理分析。

2. [任务一：基于机器学习的文本分类实践](Task1/README.md)
   包含手动实现的 Softmax 回归模型，探讨了词袋模型与 N-gram 特征对分类性能的影响。

3. [任务二：基于深度学习的文本分类进阶](Task2/README.md)
   包含 TextCNN、双向 LSTM 及 Transformer 分类器的对比，分析了预训练词向量对模型泛化的贡献。

4. [任务三：Transformer 核心架构实现与逻辑建模](Task3/README.md)
   包含编码器-解码器架构及 Decoder-only 模型的实现，探索了序列逻辑学习与语言生成能力。

## 环境配置指南

本项目使用 uv 工具进行环境依赖管理与虚拟环境配置。

在项目根目录下执行以下命令完成环境同步：
```bash
uv sync
```

执行特定任务的训练或测试脚本：
```bash
uv run python [脚本相对路径]
```

## 核心实验结论总结

### 任务零：推理引擎性能分析
实验对比表明，vLLM 框架利用分页注意力机制与连续批处理技术，在处理并发请求时，其生成吞吐量显著优于原生推理框架。

### 任务一：特征工程与线性模型
实验验证了特征稀疏性对分类性能的负面影响。在特征规模受限的情况下，单字词袋模型通过更高的覆盖率，在准确率上优于双字模型。

### 任务二：预训练初始化与架构泛化
实验数据揭示了 GloVe 预训练词向量对提升模型分类准确率的关键作用。双向 LSTM 结构通过捕捉上下文依赖，展现了较强的模型泛化能力。

### 任务三：Transformer 逻辑学习
实验证明了反转序列技巧在解决加法进位逻辑中的决定性作用。同时，词级 Tokenizer 相比字符级 Tokenizer 在处理长程语义关联时具有显著的收敛优势。
